[2026-07-10] REAL FIX: CENTER-CLUMP / SHAKING (roam re-aim bias)
================================================================
CONFIRMED by watching it run: by ~34s all 15 goblets collapsed into a
jittering clump at the exact map centre (500,350), pop still 15
(nobody ate/drank/reproduced) — they were being *dragged* inward.

ROOT CAUSE: roamToward() re-picked a fresh UNIFORM-RANDOM destination
every 4 seconds (the anti-stuck timer). A goblet rarely reaches a far
point in 4s, so it was constantly re-aiming at a new random point from
wherever it stood. The average direction toward a uniform-random point
in the rectangle is toward the rectangle's CENTROID = map centre, so
the whole population drifted to the middle and jittered there.

FIX: goblets now COMMIT to each random destination until they actually
arrive (uniform coverage, no centre bias). Re-pick happens only on:
  - arrival (within 25px), or
  - being genuinely blocked: no progress (distance-to-target not
    shrinking) for 2.5s — detected via roamBestDist / roamProgressTime.
Removed the fixed 4s timer (roamExpire) entirely. The lake-block case
is still handled, now without the centre-drift side effect.

[2026-07-10] FIX: SHAKING / CLUSTERING AT LAKE EDGES
================================================================
PROBLEM: ~25-30s in, goblets started shaking and drifting toward the
middle.
CAUSE: the tangential slide "remove inward velocity component" trick
zeroes out almost ALL velocity when the target sits directly behind a
lake (desired direction == inward normal). Goblets froze and jittered
at the lake edge with near-zero speed. Since the two lakes are at
opposite corners, goblets stuck on their inner edges piled up along
the center diagonal -> looked like "moving to the middle."

FIX:
✓ Instead of deleting the inward component (which can cancel the whole
  velocity), a goblet heading into a lake is now redirected to a
  FULL-SPEED TANGENT so it circles the edge and keeps moving.
✓ The go-around side is chosen deterministically (the tangent aligned
  with current heading), so it commits to one direction and never
  flip-flops -> no more shaking.
✓ Hard push-out and bounds clamp unchanged.

[2026-07-10] PATHFINDING FIX (no more center-clustering / starving)
================================================================
PROBLEM: goblets grouped in the middle and got stuck until they died.
CAUSES (in the old ray-based avoidance):
  - World EDGES were treated as obstacles, so goblets were constantly
    reflected inward -> everyone drifted to the center, away from the
    edge/corner bushes & lakes, and starved.
  - When a roam point sat behind a lake, the goblet oscillated at the
    lake edge, never got within 25px to pick a new point, and never
    reached the resource -> stuck forever.
  - Stuck-recovery only nudged wanderAngle, which roaming ignores, so
    roaming goblets could never break out.

FIX:
✓ Removed isObstacleAhead() / getAvoidanceDirection() entirely.
✓ moveGoblet() now walks straight to the target and handles lakes with
  TANGENTIAL SLIDING: the velocity component pointing into a lake is
  removed, so goblets smoothly slide around the edge instead of
  stopping or oscillating. A hard push-out guarantees they never end
  up inside a lake. No edge avoidance — bounds are just clamped, so
  the whole map (including edge resources) is reachable again.
✓ Lake collision is skipped while a goblet is thirsty, so it can walk
  up to the water and drink (drinking triggers at the lake edge).
✓ roamToward() now re-picks its random point when reached OR after a
  4-second timeout (roamExpire), so a blocked goblet can never orbit a
  lake forever — hungry/thirsty goblets keep walking until they find
  a resource. pickRoamPoint() also avoids choosing points inside lakes.

[2026-07-10] HEAVY-TAILED GENETIC DEVIATION
================================================================
WHAT: inheritance deviation changed from a flat uniform range to a
heavy-tailed distribution — usually kids resemble both parents, but
rarely a goblet is born radically different; the more extreme the
mutation, the rarer it is.

HOW:
✓ Added gaussianRandom() (Box-Muller standard normal): small
  deviations common, large ones increasingly rare by nature.
✓ Added deviation(amount): Gaussian core (amount ≈ 1 std dev) PLUS a
  rare "macro-mutation" (MACRO_MUTATION_CHANCE = 4%) that multiplies
  the deviation by ~4-10x (itself Gaussian-weighted, so bigger jumps
  are rarer). Result: mostly small, occasionally extreme, unbounded.
✓ inheritGenes() now uses deviation() for sight, speed, size AND all
  three color channels — so a rare mutant can look totally different
  in stats or color, not just drift slightly.
✓ Replaced the old uniform "(random*2-1)*amount" everywhere in
  inheritance. (Starting spawns unchanged.)

TUNING: raise MACRO_MUTATION_CHANCE for more frequent extremes, or
the per-stat `amount` values for a wider everyday spread.

[2026-07-10] SEARCH BEHAVIOR + MATE-ON-SIGHT
================================================================
SEARCH (matches the video's described logic):
✓ When hungry/thirsty, a goblet first checks its sight radius for a
  resource. If one is in sight -> commit to walking straight to it.
✓ If nothing is in sight, it now PICKS A RANDOM POINT and walks
  toward it (roamToward()), choosing a new random point once reached,
  and re-checking its sight radius every tick. As soon as a resource
  comes into range, it overrides the random point and heads for it.
  (Previously it just jittered in place via wander().)
✓ roamTarget cleared when entering a seeking state and when a
  resource is found, so each search starts fresh.
✓ Idle (not hungry/thirsty) still uses gentle wander().

MATING:
✓ Reworked so mating triggers whenever an eligible mate is in range,
  regardless of what the goblet was doing. The mating check now runs
  every tick (gated by canReproduce) instead of only in the idle
  branch of the state machine.
✓ Mate range = the goblet's own sight range (genes.sightRange),
  so better-sighted goblets can find mates from farther away.

[2026-07-10] METABOLISM COST (SPEED + SIZE -> HUNGRIER)
================================================================

so evolution doesn't run away to max stats for free.

CHANGES:
✓ Added metabolism(g) = (speed/SPEED_REF)*0.6 + (currentSize/SIZE_REF)*0.4

  drinking/mating.
✓ Thirst left on the flat rate for now (say the word to make speed
  cost thirst too, like the video).

[2026-07-10] COLOR NOW BLENDS (PURPLE BABIES)
================================================================
WHY: color wasn't being averaged — it was a discrete red/blue gene
that inherited from ONE parent at random, so red x blue never made
purple. (Sight/speed/size WERE already averaging correctly.)

CHANGES:
✓ Color gene is now continuous RGB instead of the string 'red'/'blue'.
  - createGenes(): starting goblets are pure red {220,30,30} or
    pure blue {30,60,220}.
  - inheritGenes(): each RGB channel = average of both parents +-20,
    so a red x blue pairing yields purple (with variation toward
    either parent), and mixed goblets blend further over generations.
✓ Skin tinting is now dynamic: getTintedSkin(color) tints the
  human-skin silhouette to any RGB and caches the result per color
  (keyed by rounded r,g,b) so each shade is only rendered once.
✓ Removed the fixed skinRed/skinBlue prebuild (buildTintedSkins).
